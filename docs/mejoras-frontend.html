Reporte de cambios · Frontend

Optimización, responsividad y limpieza del frontend

Panadería Luz Marina — mejoras de rendimiento, escalabilidad, usabilidad y consistencia en todas las páginas públicas.

Fecha: Julio 2026 Alcance: HTML, CSS, JavaScript, tooling Compatibilidad: móvil · tablet · laptop · escritorio · TV/4K

El objetivo fue lograr un frontend limpio, dinámico, funcional y adaptable a todas las pantallas, revisando el HTML semántico, las hojas de estilo, el JavaScript y la automatización de tareas, sin romper la suite de pruebas ni la línea de diseño existente. 174 pruebas siguen pasando y el linter, el formateo y la CI quedan en verde.

1 Rendimiento de carga

Se eliminó el cuello de botella del CSS y se estabilizó el renderizado de imágenes.

Rendimiento

Fin de las cadenas de @import en CSS

Cada página encadenaba 7–8 @import dentro de un único CSS, forzando descargas secuenciales (una hoja no empieza a bajar hasta que termina la anterior). Ahora las hojas base y de componentes se declaran como <link> en el <head>, por lo que el navegador las descarga en paralelo.

Antes inicio.css → @import variables → @import reset → @import header → …
Descarga en cascada, render bloqueado más tiempo.
Después <link> variables · reset · header · … · inicio
Descarga paralela, primer render más rápido.
Rendimiento · CLS

Imágenes con dimensiones intrínsecas y decodificación asíncrona

Todas las <img> del catálogo y de «Nosotros» llevan ahora width, height y decoding="async", además del loading="lazy" ya existente. Esto refuerza la reserva de espacio (menos saltos de maquetado / CLS) y libera el hilo principal durante la decodificación.

2 Consistencia y corrección de errores

Bug

Tipografías coherentes en el catálogo

catalogo.html cargaba Cormorant Garamond + Inter, fuentes que el CSS del sitio ni siquiera usa (el sistema emplea Manrope/Montserrat). Se corrigió para cargar la misma familia que el resto del sitio: se elimina una descarga inútil y se unifica la apariencia.

Bug · UX

Anti-parpadeo de tema en el catálogo

El catálogo era la única página sin theme-init.js, por lo que en modo oscuro aparecía un destello claro antes de aplicar el tema. Se añadió el script anti-FOUC como en el resto de páginas.

3 Seguridad y funcionalidad

CSP · Funcional

Fallback de imágenes compatible con la CSP

«Nosotros» usaba manejadores onerror="…" en línea que la política de seguridad del sitio (script-src 'self') bloquea en producción: el reemplazo de imágenes rotas por su emoji nunca se ejecutaba. Se trasladó la lógica a un módulo (JS/pages/nosotros.js) mediante data-img-hide y data-img-fallback, cubriendo incluso el caso de imágenes que fallan antes de que el módulo cargue.

4 Escalabilidad y responsividad

Responsive · TV/4K

Nuevo nivel para pantallas muy grandes

Se añadió un breakpoint @media (min-width: 2560px) que amplía el contenedor (--container-max: 1680px), escala la tipografía y el espaciado, y muestra una quinta columna de productos en el catálogo. Así el sitio aprovecha el ancho y se lee bien a distancia en televisores y monitores 2K/4K.

  • Móvil (≤480px): 1 columna, acciones a ancho completo.
  • Tablet (≤900px): 2 columnas.
  • Laptop/escritorio: 3–4 columnas.
  • TV/4K (≥2560px): 5 columnas, tipografía y contenedor ampliados.

5 Internacionalización (i18n)

Usabilidad · Accesibilidad

El cambio de idioma ES/EN ahora traduce el contenido visible

El botón de idioma solo cambiaba la etiqueta y <html lang>, pero el texto de las páginas seguía en español porque el marcado no tenía atributos data-i18n. Se completó el sistema existente en JS/core/i18n.js (sin dependencias externas) y se anotó todo el contenido visible de las seis páginas.

  • Diccionario ES/EN ampliado (132 claves) cubriendo navegación, hero, valores, catálogo, carrito, «Nosotros», contacto, checkout y footer.
  • Traducción de atributos con data-i18n-placeholder y data-i18n-aria-label (placeholders de formularios y etiquetas ARIA).
  • Contenido generado por JavaScript localizado con t(): feedback «Añadido / Error» del catálogo, aria-labels y pluralización del carrito, y pantalla de confirmación del checkout.
  • Evento lang:changed al cambiar de idioma; las vistas dinámicas (carrito) se re-renderizan para reflejar el nuevo idioma al instante.
  • Tests ampliados: atributos, t(), getCurrentLang() y evento.

6 Automatización y tooling

CI

Integración continua de nuevo en verde

El reporte analisis_estructurado_PLM.html rompía el paso format:check de la CI. Al ser un documento generado con su propio formato, se excluyó de Prettier vía .prettierignore (igual que IMG/), dejando lint, format:check y test en verde.
